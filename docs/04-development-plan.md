# 开发计划与验收标准

## 实现顺序

1. 生成并校验 3500 字静态数据文件。
2. 建立 React/TypeScript/Vite/PWA 工程与设计令牌。
3. 实现 IndexedDB 仓库和纯 TypeScript 掌握度领域层。
4. 实现儿童首页、快速扫描、反馈学习卡、复习和字册。
5. 实现家长报告、字库筛选、备份/恢复/清除数据。
6. 补充键盘、语音、减少动态效果和响应式布局。
7. 单元测试、生产构建、浏览器交互和视觉验收。

## 工程质量

- TypeScript 严格模式。
- 领域规则有单元测试，不依赖 DOM。
- `npm run build` 零错误。
- 核心流程没有 API Key 也能运行。
- 运行时默认不发送儿童数据。
- 所有数字来源和内容状态可追踪。

## MVP 验收

- 能在桌面与平板宽度完成：首页 → 选择 30/60/100 字 → 三档作答 → 查看反馈 → 回到今天；家长验证后可查看报告。
- 刷新或关闭浏览器后进度仍在。
- 可搜索和筛选完整 3500 字，字数自动化校验为 3500 且无重复。
- 默认提供 Kai 与 Lorik 两个档案，切换后学习事件、掌握度、复习队列和报告互不混合。
- 独立单字作答后展示“字 → 词 → 句”，公版经典句标注作者与篇名。
- 备份 JSON 可恢复到空白浏览器状态。
- AI 关闭或完全不可用时不影响学习、测评、复习与报告。
- 自报“会读”不会直接进入稳定掌握。
- 颜色不是唯一状态信号；键盘可以完成核心操作；动画遵守 `prefers-reduced-motion`。
- Service Worker 与 manifest 可使生产构建安装为 PWA；首次加载后静态应用壳可离线。

## 二期验证门槛

只有满足以下条件才接入运行时大模型：

- 监护人开关、隐私告知和数据删除路径已完成。
- 有至少 200 条人工审核的生成内容评测集。
- 输出门禁、超时回退、预算上限和审计日志已实现。
- 模型替换回归测试覆盖正确性、适龄性、字表约束和敏感内容。

教材出处与频次上线前必须完成合法来源确认、版本字段、复算脚本和人工抽检。
