# 小学诗词馆开发说明

日期：2026-07-16

## 功能定位

诗词馆是识字系统里的“答后文化语境与自由阅读”模块，不是第二套背诵打卡产品。它通过一首完整诗、一个可理解的画面和少量考据说明，让孩子在不影响快速识字主流程的前提下，看见单字如何进入词句、景物和文化记忆。

## 数据与覆盖

- 总量：75 篇；汉 2、北朝 1、唐 46、宋 20、元 1、明 1、清 4。
- 每篇字段：标题、作者、朝代、完整诗文、儿童释义、情绪、历史边界、画面依据、证据等级、图片、无障碍替代文本、作者常识和可选审核音频。
- 作者资料：45 类作者或匿名作品来源全部具备“身份、作品特点、记忆锚点”，汉乐府与北朝民歌不伪装成个人作者。
- 图片：75 幅；其中 8 幅位于 `public/images/classics/`，67 幅位于 `public/images/poems/`。
- 运行时来源：`src/data/primary-poems.ts`。
- 提示词清单生成器：`scripts/poem-image-manifest.ts`。

## 界面

- 主导航与首页均提供“诗词馆”入口；
- 可搜索诗名、作者、朝代、诗句、释义和情绪；
- 可按汉魏北朝、唐、宋、元明清筛选；
- 默认显示 12 篇，再按 12 篇展开，图片使用 `loading=lazy`；
- 详情展示完整诗文、儿童释义、诗的气质、时代与考据边界、构图理由；
- 详情顶部提供“听完整朗读/停止朗读”，关闭详情或按 Escape 会立即停止；
- 作者卡放在儿童释义之后，只保留中国学生最应知道的身份、风格和代表作，不堆砌生卒年；
- 自动抽取诗中常用汉字作为“认一认”提示，并明确不写入掌握度；
- 关闭弹窗支持背景点击、关闭按钮和 Escape 键。

## 不干扰识字的约束

1. 不在孩子独立回想汉字之前出现诗词提示；
2. 不把打开、朗读或浏览诗词作为识字正确证据；
3. 不设置连续打卡、排行或强制背诵任务；
4. 不用大模型在线向儿童自由对话；
5. 图片与扩展说明可跳过，核心扫描和复习离线可用。

## 朗读方案与选型

首版使用设备内置 `zh-CN` 普通话语音：优先标准普通话音色，排除 Eddy、Grandma 等趣味角色音色，语速为 0.84；没有命中优选名字时回落到设备的普通话默认音色。这个方案在 iPad/Safari 和 Mac 上无需账号、无需联网、不会暴露 API 密钥，也不上传儿童数据。

运行时同时保留每首诗的可选 `narrationAudio` 字段。一旦完成高质量音频审听，播放器会优先播放静态音频，设备语音只作兜底。正式批量音频推荐在发布前预生成，不在儿童每次点击时调用模型：

1. 首选候选是 Azure Speech 的普通话神经音色，因为官方列表提供中文专用音色及 `poetry-reading` 风格；
2. OpenAI `gpt-4o-mini-tts` 可按提示控制语气并支持中文，但官方也说明当前内置音色仍主要针对英语优化，因此必须先做中文古诗盲听对比，不能直接批量上线；
3. 每首音频需人工核对多音字、古音/今读选择、停连、作者署名和异文，再存为 `public/audio/poems/<slug>.mp3`；
4. 页面应明确说明合成语音，不把“听过”写入识字掌握度。

参考：[Azure Speech 语言与音色支持](https://learn.microsoft.com/en-us/azure/ai-services/speech-service/language-support)、[OpenAI Text-to-Speech 指南](https://developers.openai.com/api/docs/guides/text-to-speech)。

## 验收

- 数据测试确认 75 个连续 ID、75 个唯一 slug、每篇完整字段与合法证据等级；
- 数据测试确认 75 篇均能解析到作者/来源常识卡，朗读文本依次包含标题、朝代作者和完整诗句；
- 资源测试确认所有 75 个图片路径真实存在；
- 67 幅新图统一为 1280×853 JPEG，总量约 25MB；
- PWA 图片缓存上限 80 条，经典图和诗词馆图共用 `poetry-scenes-v2` 缓存；
- 桌面和手机断点均可使用，手机端诗词详情改为单列；
- iPad 1024×768 横屏和 768×1024 竖屏均在首屏显示朗读按钮；
- `npm test` 与 `npm run build` 必须同时通过后才可部署。
